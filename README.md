# Reaction Roulette

A local-first meme/reaction GIF search and clipboard tool. Paste a Tenor/Giphy/Discord URL, search by natural language later, click to copy the URL back into Discord.

## Architecture

```
┌─────────────────────────────────────────────────────────┐
│  apps/windows_app/        — Flutter desktop UI (Forui)  │
│  ├─ lib/widgets/          — sidebar, search bar, grid   │
│  ├─ lib/clipboard_menu.dart — right-click copy options  │
│  ├─ lib/storage_wizard.dart — first-launch path picker  │
│  ├─ lib/inference_impl.dart — ONNX CLIP + OCR impls     │
│  └─ integration_test/     — ONNX model validation       │
├─────────────────────────────────────────────────────────┤
│  packages/meme_collector_core/  — pure Dart backend     │
│  ├─ lib/models.dart       — Reaction entity + JSON      │
│  ├─ lib/storage.dart      — metadata.json + atomic save │
│  ├─ lib/inference.dart    — abstract embedder interfaces│
│  ├─ lib/search.dart       — SIMD vector index + RRF     │  (TODO)
│  ├─ lib/ingest.dart       — URL parser + ffmpeg + pipe  │  (TODO)
│  └─ lib/worker.dart       — isolates + message protocol │  (TODO)
├─────────────────────────────────────────────────────────┤
│  scripts/                                                  │
│  ├─ download_models.dart  — fetches HF model files      │
│  └─ validation_report.json — generated by validation    │
├─────────────────────────────────────────────────────────┤
│  assets/                                                   │
│  ├─ models/               — ONNX + model2vec (~478 MB)  │
│  └─ bin/                  — ffmpeg + vcredist (~81 MB)  │
└─────────────────────────────────────────────────────────┘
```

Core is pure Dart (no Flutter deps) so it can later be imported by a
`dart_frog` server for the future Android/remote client. The app provides
Flutter-dependent concrete implementations of the inference interfaces
(CLIP via ONNX Runtime, OCR via ONNX Runtime).

## Setup

### 1. Prerequisites

- Flutter 3.44+ (beta channel OK, matches SDK constraint)
- Rust toolchain (for model2vec Native Assets build)
- Windows 10 1607+ (for atomic file rename via MoveFileExW)

### 2. Install dependencies

```bash
cd meme_collector_project
flutter pub get
```

### 3. Download model files (~478 MB)

```bash
dart run scripts/download_models.dart
```

Downloads to `assets/models/`:
- `potion-base-32M/` — model2vec text embeddings (~150 MB)
- `clip_text_int8.onnx` — CLIP text tower, INT8 (64 MB)
- `clip_vision_q4.onnx` — CLIP vision tower, Q4 weight-only quant (64 MB)
  - **NOT int8** — the int8 variant uses `ConvInteger` ops that the CPU EP
    doesn't support. Q4 is weight-only 4-bit quant, dequantizes to FP32
    at load time, uses standard Conv ops.
- `clip_tokenizer.json` — CLIP BPE tokenizer spec
- `ppocr_det.onnx` — PP-OCRv5 detection (84 MB)
- `ppocr_rec.onnx` — PP-OCRv5 recognition (~8 MB)
- `ppocr_dict.txt` — character dictionary

### 4. Validate models (GATE — do this before anything else)

```bash
cd apps/windows_app
flutter test integration_test/validate_models_test.dart -d windows
```

This loads each ONNX model, runs synthetic input through it, and verifies
output shapes. If any model fails, fix or find an alternative before
writing real inference code.

Report saved to `scripts/validation_report.json`.

### 5. Download bundled binaries (ffmpeg + vcredist)

Not automated yet. Manually place:
- `ffmpeg.exe` (~80 MB) from https://www.gyan.dev/ffmpeg/builds/ → `assets/bin/`
- `vcruntime140.dll` + `msvcp140.dll` from VCRedist → `assets/bin/vcredist/`

### 6. Run the app

```bash
cd apps/windows_app
flutter run -d windows
```

On first launch, the storage wizard asks where to put your library.

## Storage layout

User-chosen root (default: `./storage/` next to the exe):

```
storage/
├── metadata.json                    — source of truth (URLs, tags, usage)
├── downloads/{id}.{ext}             — original files (pinned + recent)
├── thumbnails_static/{id}.webp      — static WebP previews (always)
├── thumbnails_animated/{id}.webp    — animated previews (if enabled)
├── embeddings_text/{producer}__{version}/{id}.f32.bin
├── embeddings_image/{producer}__{version}/{id}.f32.bin
└── ocr/{id}.txt
```

The `.sqlite`-free design: metadata.json is small (~5 MB for 10k entries),
vectors live in memory at runtime (~20 MB for 10k × 512-dim), derivative
files on disk are the source of truth for everything else.
